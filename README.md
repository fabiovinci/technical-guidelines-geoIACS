## 🚀 How to Contribute

- **Discussions**: Participate in the [dedicated discussions section](https://github.com/orgs/geoIACS/discussions). To facilitate the understanding of the distinct geoIACS implementation requirements from the MS,  participants to discussions are encouraged to mention their MS/organisation (unless this is already clear from their GitHub username).
- **Change Proposals**: Use the [issue tracker](https://github.com/geoIACS/technical-guidelines/issues) to submit change proposals to the TG (opening a New issue and using the dedicated template).

## 📋 Release Management

This repository uses a **release-per-branch** model:

- **`main`** - Contains the latest stable version of the Technical Guidelines
  - Automatically published via [GitHub Pages](https://geoiacs.github.io/technical-guidelines/)
  - View the live documentation at [geoIACS Technical Guidelines](https://geoIACS.github.io/technical-guidelines/tg)

- **Release branches** (`release/1.0`, `release/2.0`, etc.) - Archive previous versions
  - Available for historical reference and maintenance
  - Each release is tagged with its version number (e.g., `v1.0`, `v2.0`)

## 📖 About the Guidelines

The geoIACS Technical Guidelines describe detailed implementation aspects and relations with existing standards, technologies, and practices.
